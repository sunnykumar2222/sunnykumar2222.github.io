University Website Testing

Hands-on quality assurance on a live, in-production website — finding the problems real users would have hit first.

Ask me about it

Organization

Dr. C.V. Raman University, Vaishali

Scope

Functional, usability, performance QA

Method

Agile, full SDLC cycle

The assignment

The university's official website was in active development. My job was to test it the way a real visitor would — and find everything that would break before it went live.

What I did

  • Ran comprehensive functional, usability, and performance testing across the site's active development cycle.
  • Validated UI/UX, navigation flow, and responsiveness across multiple real-world usage scenarios.
  • Documented over 60 distinct bugs and inconsistencies — functional errors, UI defects, and UX gaps — with clear repro steps.
  • Worked directly inside the core dev team's Agile workflow, turning every bug report into specific, actionable feedback.
